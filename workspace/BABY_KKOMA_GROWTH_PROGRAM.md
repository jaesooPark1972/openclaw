# 아기꼬마 육아성장 프로그램 (실행판)

시작일: 2026-02-20
목표: 작은 자원(8GB 환경)으로 추론 정확도, 문서 이해력, 실행 안정성을 7일 단위로 끌어올린다.

## 0) 매일 공통 루틴 (20~30분)
1. 실전 질문 10개 수집
2. 좋은 답변/나쁜 답변 각각 5개 기록
3. 오답 원인 태깅 (사실오류/형식오류/누락/과장)
4. 교정 프롬프트 1개 추가

## 1) 7일 스프린트
### Day 1 — 기준선 측정
- 테스트셋 30문항 실행
- 지표 기록: 정답률, 형식준수율, 평균응답시간

### Day 2 — 데이터 품질 정리
- good_qa 50개, bad_qa 50개 축적
- 포맷 통일: 질문 / 의도 / 정답 / 오답 / 이유

### Day 3 — 출력 규칙 고정
- 출력 템플릿 강제: 요약 → 근거 → 다음행동
- 숫자/날짜/단위 검증 규칙 적용

### Day 4 — 2단계 추론
- 1차: 아기꼬마 초안
- 2차: 검증기(규칙 기반) 자동 교정

### Day 5 — 메모리 결합
- Upstash 단기 컨텍스트 3개
- LanceDB 장기 사례 3개
- 답변 전 6개 참고 후 출력

### Day 6 — 실패학습 집중
- 오답 유형별 교정 프롬프트 자동 삽입
- 반복 오류 Top3 제거

### Day 7 — 재측정/리팩토링
- Day1 테스트셋 재실행
- 점수 비교 후 규칙/프롬프트 1회 개선

## 2) 모델 라우팅 (고정)
- 코딩: qwen2.5-coder:7b
- 문서 이미지: glm-ocr:latest → qwen3 후처리
- 일반 질의/요약: qwen3:latest

## 3) 성공 기준 (한 스프린트)
- 정답률 +10%p
- 형식준수율 90%+
- 반복 오류 30% 감소

## 4) 실행 규칙
- 무거운 모델 상시 로드 금지
- 기록은 반드시 남긴다 (누적 성장 우선)
- 주인님 지시 문체/톤 우선 유지
