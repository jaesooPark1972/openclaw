# OpenClaw 운영 파일 맵 (실전판)

| File | Purpose |
|---|---|
| `SOUL.md` | 에이전트의 성격, 말투, 행동 원칙 |
| `IDENTITY.md` | 에이전트 정체성(이름/역할/캐릭터) |
| `USER.md` | 사용자 정보, 선호, 목표, 컨텍스트 |
| `TOOLS.md` | 로컬 환경 메모(경로, 장치, 실무 팁) |
| `AGENTS.md` | 운영 규칙, 실행 정책, 보안/행동 가이드 |
| `MEMORY.md` | 메모리 시스템 구조(장기 기억 설계) |
| `WORKFLOW_AUTO.md` | 자동 실행 워크플로우/우선순위/일일 루틴 |
| `memory/YYYY-MM-DD.md` | 날짜별 지속 메모(당일 결정/이슈/후속 과제) |

---

## 빠른 운영 원칙
1. 정책/행동 기준은 `AGENTS.md`, 성격/톤은 `SOUL.md`를 우선.
2. 사용자 맞춤 응대는 `USER.md`를 기준으로 유지.
3. 자동화 루틴 변경 시 `WORKFLOW_AUTO.md` 먼저 갱신.
4. 중요한 결정/이슈는 반드시 `memory/YYYY-MM-DD.md`에 기록.
5. 장기 구조 변경은 `MEMORY.md`와 함께 맞춰서 반영.
